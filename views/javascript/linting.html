JavaScript Linting

  • npm install eslint and babel then copy the linting rules from fs-javascript/linters repo ✔ 
          
        npm install --save-dev \
        eslint \
        babel-eslint \
        fs-webdev/fs-javascript && \
        cp ./node_modules/fs-javascript/linters/eslint-default.txt .eslintrc && \
        cp ./node_modules/fs-javascript/linters/.eslintignore .eslintignore
  • Setup ESLint plugin for Editor (linter-eslint for Atom) ✔
  • Have a .eslintignore to avoid checking things like node_modules ✔
  • Ran linter and discovered that the fs lint config states we should use single quotes for strings ✔

Notes:

  • ESLint will use the closest config file and then go up through the tree back to root applying configs. To prevent this you can put "root": true in your config file.
